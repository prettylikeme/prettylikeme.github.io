使用 CSS 调整透明度

  • CSS

  1. 使用 opacity

    值 描述
    value 规定不透明度。从 0.0 （完全透明）到 1.0 （完全不透明）
    inherit 应该从父元素继承

    子元素会继承父元素的透明属性

  2. 使用 background-color

    值 描述
    color_name 规定颜色值为颜色名称的背景颜色（比如 red ）。
    hex_number 规定颜色值为十六进制值的背景颜色（比如 #ff0000 ）。
    rgb_number 规定颜色值为 rgb 代码的背景颜色（比如 rgb(255,0,0) ）。
    transparent 默认。背景颜色为透明。
    inherit 规定应该从父元素继承 background-color 属性的设置。

    我们可以使用background-color: rgba(255, 255, 255, 0.5)来设置元素的透明度

    不会影响子元素的透明度

数组

部分遍历相关函数：

  • every ：所有元素经过回调返回值都为true，则函数返回值为 true ，否则为 false
  • some ：只要有元素经过回调返回值为true，则函数返回值为 true ，否则为 false
  • includes ：包含指定的元素时返回 true
  • map ：每个元素在回调函数中的返回值，替换原来的值
  • find ：查找元素
  • findIndex ：查找元素的索引值
  • filter ：回调函数返回值为 false 的元素将被过滤
  • reduce ：用于求值，将数组中的多个元素，通过回调计算为一个值返回

模块概念

Nodejs 实现模块化采用的标准是 CommonJS

  • 使用 require 来加载包
    • 在 node 中变量没有全局作用域，只有模块（文件）作用域
    • require 只能加载被导出的对象、函数或值
  • 使用 exports 导出对象
    • 每个模块自带一个 module 对象， module 对象中有一个 exports 对象
    • 每个模块也自带一个 exports 对象，是 module.exports 的引用

模块的种类

  • 核心模块
    • 核心模块在 Nodejs 安装后就自带，可直接加载使用，模块名不带 .js
    • 例： require('path')
  • 第三方模块
    • 需要通过 npm 进行安装，安装后即可像使用核心模块一样使用，模块名不带 .js
    • 例： require('express')
  • 自定义模块
    • 用户自定义的 js 模块文件，使用时需写相对路径，模块名可带可不带 .js
    • 例： require('./func')

Node 中的其他成员

在每个模块中，都有 4 个特殊的成员

  • require ：模块导入方法
  • exports ：模块导出对象
  • __dirname ：当前文件所属目录的绝对路径
  • __filename ：当前文件的绝对路径

Node 中文件操作的相对路径是相对 Node 的执行环境目录的，而不是代码文件所属目录。 require 方法除外。
