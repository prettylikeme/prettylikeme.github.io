VueX

2018-07-19
  • Html
  • Vue.js

VueX 是专门为 vue.js 设计的一个状态管理器插件。
简单的理解，就是 Vue 中一个管理全局变量的工具。

状态，简单的理解即是变量或者数据。

在 Vue 中使用 VueX

states

相当于 Vue 组件内的 data 属性，用于保存状态。
Vue 推荐在应用内只使用一个 VueX 对象进行状态保存，方便管理和维护。

mutations

  • 唯一用于修改 states 的途径，就是通过提交 mutation ，不要直接去修改状态的值
  • 对于 mutations 中的方法，建议使用常量来定义其函数名
  • mutations 中的函数必须是同步的

actions

当涉及到有一些异步操作需要修改状态时，就需要用到 action 。

modules

项目过大时，需划分模块，针对不同模块，可将 store 按模块划分。

getters

相当于 Vue 组件内的 computed 属性。

getters 中函数的参数：

  • state ：用此参数可以获取对应同层级 state 中的值
  • getters ：可以借此参数，引用对应 getters 中的其他 getter
  • rootState ：在模块中时，用此参数可以获取根 state 中的值

组件间的通信

父组件向子组件传递

子组件使用 props 属性来接受数据，父组件在子组件实例上使用 v-bind 向子组件传递值。

子组件向父组件传递

子组件通过 $emit 发送自定义事件，父组件使用 v-on 来监听指定的事件，以此来完成通信。

插槽 slot

插槽为组件预留一个或多个接口，方便组件拓展。

  • 匿名插槽
    • 在组件模板中使用 <slot></slot> 即可定义一个插槽，在组件标签内填入的内容即会替换到所有匿名插槽所在位置。
  • 插槽的默认值
    • 在定义插槽时，可以在内部填入内容，该内容为插槽的默认内容。
  • 具名插槽
    • 插槽也可以有 name 属性，使用时也需要在组件内的标签上使用 slot 属性。
  • 作用域插槽
    • 变量作用域：变量属于该变量所在模板所属的组件
    • 通过在插槽替换内容中添加 slot-scope="slot" 属性，即可通过 slot.<dataname> 访问插槽上绑定的数据 <slot :<dataname>="<data>"></slot>

key 属性

Vue 在渲染时，为了优化浏览器的性能，会尽可能的复用已经存在在页面上的元素。
因此会存在一些问题：

  • 使用 v-if 系列指令时，如存在 input 输入框时，当其内容不为空时进行切换显示，其内容不会被清空
  • 使用 v-for 循环遍历数组，在数组头部或中间插入元素时，它不会直接在插入的位置直接插入标签值，而是会修改插入位置的值为插入的值，在末尾添加一个元素，并相应修改后面的每个元素的值。如此会影响浏览器的渲染效率。

因此 Vue.js 推荐在遍历时，为每个元素加上一个 key 属性。（ key 的值不能使用 index ，因为 index 会随着元素的插入删除也会变化。）

  3. 优化配置

    1. 开发环境性能优化

      1. HMR ： devServer: { hot: true }

      2. source-map ： devtool: 'source-map'

        [inline-|hidden-|eval-][nosource-][cheap-[module-]]source-map

        • 开发环境推荐： eval-source-map
        • 生产环境推荐： eval-cheap-module-source-map

    2. 生产环境性能优化

      1. oneOf ： module: { rules: [ oneOf: [ ... ]] }
      2. babel 缓存 ： rules:[{ loader: 'babel-loader', options: { cacheDirectory: true }}]
      3. 资源缓存：在文件名中使用 hash 值
        • hash ：每次打包时会生成一个唯一的 hash 值。会产生一个小改动就全部缓存失效的问题
        • chunkhash ：根据 chunk 生成的 hash 值。如果打包来源于同一个 chunk ，也会产生用打包生成的 hash 值一样的问题
        • contenthash ：根据文件内容生成的 hash 值。不同文件 hash 一定不同
      4. 去除无用代码： tree shaking 。代码中使用 ES6 模块化，且 webpack 的打包模式是 production 时就会自动开启
        • package.json 中配置 "sideEffects": false 所有代码都没有副作用，可能会造成 css / @babel/polyfill 问会被干掉
        • 使用 "sideEffects": ["*.css"] ，就能避免 css 等文件被去除
      5. js 代码分割：
        • entry 配置为对象（ key 随意， val 为文件路径）， output 的 filename 中加入 [name] （ entry 里对象的 key 值）属性用以分辨输出文件。不灵活
        • 配置 splitChunks ，能将 node_module 中代码单独打包一个 chunk 输出，也能自动分析多入口 chunk 中有没有公共的文件，如果有会打包成一个单独的 chunk
        • 通过 import(/* webpackChunkName: 'test' */'./test') 动态导入语法，可以将某个文件单独打包，通过注释给文件取名
      6. js 文件懒加载：运用代码分割第三种方式的特点实现异步加载。
        • 预加载：可以在其中配置 webpackPrefetch: true 对文件进行预加载。兼容性差
      7. PWA ： service-worker
      8. 多进程打包： thread-loader ，一般用在花时间比较长的地方，如 babel-loader 中。
      9. externals ：配置 externals: { jquery: 'jQuery' } ，忽略库名： npm 包名
      10. dll ： 、 webpack.DllPlugin

常用 loader 与 plugin 及其他工具

  • loader
    • style-loader ：将 js 模块中的样式加载到引入 js 文件的 head 中
    • css-loader ：将 css 文件打包成 js 模块（ commonJS 模块）
    • less-loader ：将 less 文件编译成 css 文件，需要安装 less
    • sass-loader ：将 sass 文件编译成 css 文件
    • file-loader ：打包其他文件
    • url-loader ：样式文件中的 url 路径打包工具，依赖 file-loader
    • html-loader ：打包 html 中的图片资源等路径
    • postcss-loader ：用来配置浏览器的 css 兼容，与 postcss-preset-env 一起使用
    • eslint-loader ：语法检查，需要安装 eslint
    • babel-loader ： js 的兼容性处理，需要安装 @babel/core ，检查配置：
      1. 基本转换：安装 @babel/preset-env ，这种做法不能转换高级语法，如 Promise 等
      2. 全部语法：安装 @babel/polyfill ，并在 js 代码中引入： import '@babel/polyfill' ，这种做法转换后的 js 代码过于庞大
      3. 自定义转换：安装 core-js 并配置，这种做法最好，转换后文件体积也会变小
  • plugin
    • 插件都是使用 new 关键字创建即可
    • html-webpack-plugin ：为 html 文件自动引入打包的所有资源
    • mini-css-extract-plugin ：将 css 文件单独从 js 中提取出来，需其 loader 属性替换 style-loader
    • postcss-preset-env ：帮助 postcss-loader 找到 package.json 文件中的 browserslist 配置
    • optimize-css-assets-webpack-plugin ：压缩 css 文件
  • 其他
    • webpack-dev-server ：webpack 自动化工具，只在内存中打包，不会有文件输出
