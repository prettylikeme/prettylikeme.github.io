数组

部分遍历相关函数：

  • every ：所有元素经过回调返回值都为true，则函数返回值为 true ，否则为 false
  • some ：只要有元素经过回调返回值为true，则函数返回值为 true ，否则为 false
  • includes ：包含指定的元素时返回 true
  • map ：每个元素在回调函数中的返回值，替换原来的值
  • find ：查找元素
  • findIndex ：查找元素的索引值
  • filter ：回调函数返回值为 false 的元素将被过滤
  • reduce ：用于求值，将数组中的多个元素，通过回调计算为一个值返回

函数

call 、 apply 和 bind

这三个函数都是用来重定义函数体内的 this 指向的，只是在用法上不一样：

  • call ： func.call(obj, arg1, arg2)
  • apply ： func.apply(obj, [arg1, arg2])
  • bind ： func.bind(obj, arg1, arg2)

模块概念

Nodejs 实现模块化采用的标准是 CommonJS

  • 使用 require 来加载包
    • 在 node 中变量没有全局作用域，只有模块（文件）作用域
    • require 只能加载被导出的对象、函数或值
  • 使用 exports 导出对象
    • 每个模块自带一个 module 对象， module 对象中有一个 exports 对象
    • 每个模块也自带一个 exports 对象，是 module.exports 的引用

模块的种类

  • 核心模块
    • 核心模块在 Nodejs 安装后就自带，可直接加载使用，模块名不带 .js
    • 例： require('path')
  • 第三方模块
    • 需要通过 npm 进行安装，安装后即可像使用核心模块一样使用，模块名不带 .js
    • 例： require('express')
  • 自定义模块
    • 用户自定义的 js 模块文件，使用时需写相对路径，模块名可带可不带 .js
    • 例： require('./func')

Node 中的其他成员

在每个模块中，都有 4 个特殊的成员

  • require ：模块导入方法

Node 中文件操作的相对路径是相对 Node 的执行环境目录的，而不是代码文件所属目录。 require 方法除外。

VueX

2018-07-19
  • Html
  • Vue.js

VueX 是专门为 vue.js 设计的一个状态管理器插件。
简单的理解，就是 Vue 中一个管理全局变量的工具。

状态，简单的理解即是变量或者数据。

在 Vue 中使用 VueX

states

相当于 Vue 组件内的 data 属性，用于保存状态。
Vue 推荐在应用内只使用一个 VueX 对象进行状态保存，方便管理和维护。

mutations

  • 唯一用于修改 states 的途径，就是通过提交 mutation ，不要直接去修改状态的值
  • 对于 mutations 中的方法，建议使用常量来定义其函数名
  • mutations 中的函数必须是同步的

actions

当涉及到有一些异步操作需要修改状态时，就需要用到 action 。

modules

项目过大时，需划分模块，针对不同模块，可将 store 按模块划分。

getters

相当于 Vue 组件内的 computed 属性。

getters 中函数的参数：

  • state ：用此参数可以获取对应同层级 state 中的值
  • getters ：可以借此参数，引用对应 getters 中的其他 getter
  • rootState ：在模块中时，用此参数可以获取根 state 中的值

v-on

  1. 事件对象
    1. 绑定的函数有一个或多个参数时，调用时省略参数列表，将把 event 事件对象作为第一个参数传递给函数
    2. 绑定的函数不需要参数时，调用时可省略参数列表，实际可以理解为同上一条
    3. 绑定的函数有多个参数时，有参数列表，且需要 event 事件对象时，可使用 $event 将事件对象传递给函数
  2. 修饰符

组件间的通信

父组件向子组件传递

子组件使用 props 属性来接受数据，父组件在子组件实例上使用 v-bind 向子组件传递值。

子组件向父组件传递

子组件通过 $emit 发送自定义事件，父组件使用 v-on 来监听指定的事件，以此来完成通信。

插槽 slot

插槽为组件预留一个或多个接口，方便组件拓展。

  • 匿名插槽
    • 在组件模板中使用 <slot></slot> 即可定义一个插槽，在组件标签内填入的内容即会替换到所有匿名插槽所在位置。
  • 插槽的默认值
    • 在定义插槽时，可以在内部填入内容，该内容为插槽的默认内容。
  • 具名插槽
    • 插槽也可以有 name 属性，使用时也需要在组件内的标签上使用 slot 属性。
  • 作用域插槽
    • 变量作用域：变量属于该变量所在模板所属的组件
    • 通过在插槽替换内容中添加 slot-scope="slot" 属性，即可通过 slot.<dataname> 访问插槽上绑定的数据 <slot :<dataname>="<data>"></slot>
