key 属性

Vue 在渲染时，为了优化浏览器的性能，会尽可能的复用已经存在在页面上的元素。
因此会存在一些问题：

  • 使用 v-if 系列指令时，如存在 input 输入框时，当其内容不为空时进行切换显示，其内容不会被清空
  • 使用 v-for 循环遍历数组，在数组头部或中间插入元素时，它不会直接在插入的位置直接插入标签值，而是会修改插入位置的值为插入的值，在末尾添加一个元素，并相应修改后面的每个元素的值。如此会影响浏览器的渲染效率。

因此 Vue.js 推荐在遍历时，为每个元素加上一个 key 属性。（ key 的值不能使用 index ，因为 index 会随着元素的插入删除也会变化。）

以前有人问我，导入的时候有没有大括号的区别是什么。下面是我在工作中的总结：

  1. 当用export default people导出时，就用 import people 导入（不带大括号）
  2. 一个文件里，有且只能有一个export default。但可以有多个export。
  3. 当用export name 时，就用import { name }导入（记得带上大括号）
  4. 当一个文件里，既有一个export default people, 又有多个export name 或者 export age时，导入就用 import people, { name, age }
  5. 当一个文件里出现n多个 export 导出很多模块，导入时除了一个一个导入，也可以用import * as example

Generators

生成器（ generator）是能返回一个迭代器的函数。生成器函数也是一种函数，最直观的表现就是比普通的function多了个星号*，在其函数体内可以使用yield关键字,有意思的是函数会在每个yield后暂停。

这里生活中有一个比较形象的例子。咱们到银行办理业务时候都得向大厅的机器取一张排队号。你拿到你的排队号，机器并不会自动为你再出下一张票。也就是说取票机“暂停”住了，直到下一个人再次唤起才会继续吐票。

OK。说说迭代器。当你调用一个generator时，它将返回一个迭代器对象。这个迭代器对象拥有一个叫做next的方法来帮助你重启generator函数并得到下一个值。next方法不仅返回值，它返回的对象具有两个属性：done和value。value是你获得的值，done用来表明你的generator是否已经停止提供值。继续用刚刚取票的例子，每张排队号就是这里的value，打印票的纸是否用完就这是这里的done。

那生成器和迭代器又有什么用处呢？

围绕着生成器的许多兴奋点都与异步编程直接相关。异步调用对于我们来说是很困难的事，我们的函数并不会等待异步调用完再执行，你可能会想到用回调函数，（当然还有其他方案比如Promise比如Async/await）。

生成器可以让我们的代码进行等待。就不用嵌套的回调函数。使用generator可以确保当异步调用在我们的generator函数运行一下行代码之前完成时暂停函数的执行。

let

ES6 新增了 let 命令，用来声明变量。它的用法类似于 var ，但是所声明的变量，只在 let 命令所在的代码块内有效。

作用域

使用 let 申明的变量，只在 let 命令所在的代码块内有效。

PS: for 循环有一个特别之处，就是设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域

不存在变量提升

let 所声明的变量一定要在声明后使用，否则报错。

暂时性死区

只要块级作用域内存在 let 命令，它所声明的变量就“绑定”（binding）这个区域，不再受外部的影响。

const 同理

变量不允许重复声明

const

声明一个只读的常量。

  • 一旦声明，常量的值就不能改变；
  • 一旦声明变量，就必须立即初始化，不能留到以后赋值；
  • 只声明不赋值，就会报错。

和 let 一样：

  • 只在声明所在的块级作用域内有效；
  • 声明的常量不提升；
  • 同样存在暂时性死区；
  • 不可重复声明。

const本质

const 实际上保证的，并不是变量的值不得改动，而是变量指向的那个内存地址所保存的数据不得改动。对于简单类型的数据（数值、字符串、布尔值），值就保存在变量指向的那个内存地址，因此等同于常量。但对于复合类型的数据（主要是对象和数组），变量指向的内存地址，保存的只是一个指向实际数据的指针， const 只能保证这个指针是固定的（即总是指向另一个固定的地址），至于它指向的数据结构是不是可变的，就完全不能控制了。因此，将一个对象声明为常量必须非常小心。

块级作用域

ES5 只有全局作用域和函数作用域，没有块级作用域。

在 ES6 中， let 实际上为 JavaScript 新增了块级作用域。

  • 块级作用域必须有大括号；
  • 允许块级作用域的任意嵌套；
  • 内层作用域可以定义外层作用域的同名变量；
  • 允许在块级作用域之中声明函数。
    • 但应该尽量避免。如果确实需要，也应该写成函数表达式，而不是函数声明语句。

顶层对象的属性

  • var 命令和 function 命令声明的全局变量，依旧是顶层对象的属性；
  • let 命令、 const 命令、 class 命令声明的全局变量，不属于顶层对象的属性。

变量的解构赋值

ES6 允许按照一定模式，从数组和对象中提取值，对变量进行赋值，这被称为解构。
本质上，这种写法属于“模式匹配”，只要等号两边的模式相同，左边的变量就会被赋予对应的值。

Promise

2018-07-19
  • Html
  • ES6
  • Promise

Promise 是 ES6 中新增的一种容器。
在 ES6 之前，为了保证异步方法的先后执行顺序，会嵌套很多异步代码，这种状态叫做回调地狱（ callback hell ）。
Promise 就是为了解决这种编程方式的。

Promise 有三种状态：

  • pending ：表示执行中
  • resolved ：表示已解决
  • rejected ：表示失败

pending 状态能变成 resolved 或 rejected 中的一种，且不能二次改变

基本语法

Promise 对象包装一个方法，两个参数也是方法。

  • then
  • catch
  • all

在容器外调用时， then 方法内部的第一个函数，即是 Promise 对象的 resolve 函数；第二个函数，即是 Promise 对象的 reject 函数。

then 方法的反回值如果是 Promise 对象，则其后面的 then 方法的函数参数会作为该 Promise 对象的 resolve 和 reject 方法。
否则其作为 then 方法中的函数的参数。

模板字符串

模板字符串是增强版的字符串，用反引号（ ` ）标识。
它可以当作普通字符串使用，也可以用来定义多行字符串，或者在字符串中嵌入变量。

  • 模板字符串中嵌入变量，需要将变量名写在 ${} 之中。

  • 模板字符串之中还能调用函数。

  • 如果大括号中的值不是字符串，将按照一般的规则转为字符串。比如，大括号中是一个对象，将默认调用对象的 toString 方法。

  • 如果模板字符串中的变量没有声明，将报错。

  • 模板字符串甚至还能嵌套，使用 ${} 。

  • 如果需要引用模板字符串本身，在需要时执行，可以写成函数。
